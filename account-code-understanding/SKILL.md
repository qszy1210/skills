# 科目编码与名称理解

## 触发条件

当用户提供包含**科目编码**或**科目名称**的账簿数据时使用，包括但不限于：
- 科目余额表
- 总账 / 明细账 / 辅助账
- 任何使用 `编码\名称路径` 格式的财务数据

典型场景：需要理解科目的层级关系、上下级归属、分类含义，或需要从编码/名称中提取结构化信息。

---

## 编码规则

### 层级结构

一级科目编码为 **4位数字**，每下一级增加 **2位**：

| 层级 | 编码位数 | 示例 | 说明 |
|------|---------|------|------|
| 一级 | 4位 | `1002` | 存放中央银行款项 |
| 二级 | 6位 | `100201` | 存放中央银行准备金存款 |
| 三级 | 8位 | `10020101` | 更细分科目 |
| 四级 | 10位 | `2211010201` | 应付常规绩效工资 |

### 层级计算

```
level = (len(code) - 4) // 2 + 1
```

- `1002` → level 1
- `100201` → level 2
- `10110103` → level 3
- `6111010201` → level 5

### 父级编码

当前编码去掉末尾2位即为父级编码：

```
parent_code = code[:-2]
```

- `100201` → 父级 `1002`
- `10110103` → 父级 `101101` → 祖父级 `1011`

## 名称规则

### 格式

科目名称为**全路径格式**，以反斜杠 `\` 分隔：

```
编码\一级科目名\二级科目名\...\当前科目名
```

第一段为编码本身（部分场景下可能省略），后续各段依次为从一级到当前级的科目名称。

### 解析规则

**当前科目的实际名称 = 最后一段**

示例解析：

| 原始值 | 编码 | 层级 | 一级名 | 当前名（实际名称） |
|--------|------|------|--------|------------------|
| `1002\存放中央银行款项` | 1002 | L1 | 存放中央银行款项 | 存放中央银行款项 |
| `100201\存放中央银行款项\存放中央银行准备金存款` | 100201 | L2 | 存放中央银行款项 | 存放中央银行准备金存款 |
| `10110103\存放同业款项\存放银行业同业活期\存放政策性银行同业活期款项` | 10110103 | L3 | 存放同业款项 | 存放政策性银行同业活期款项 |
| `6111010201\投资收益\金融资产利息收入\交易性金融资产利息收入\交易性金融资产利息收入-国债` | 6111010201 | L5 | 投资收益 | 交易性金融资产利息收入-国债 |

### 仅有名称（无编码前缀）时

如果数据中只有名称路径而无编码前缀：

- 按 `\` 分隔后，**段数 = 层级**
- 最后一段为当前科目名
- 前面各段为祖先路径

示例：`存放中央银行款项\存放中央银行准备金存款` → 2段 = 二级科目，当前名为"存放中央银行准备金存款"。

### 判断第一段是编码还是名称

- 第一段**全部为数字** → 视为编码，层级由编码长度决定
- 第一段**包含中文/字母** → 视为名称，层级由段数决定

## 上下级判定

### 按编码判定

A 是 B 的**直接父级** ⟺ 同时满足：

1. B 以 A 开头（`B.startswith(A)`）
2. B 比 A 长恰好2位（`len(B) == len(A) + 2`）

示例：
- `1002` 是 `100201` 的父级 ✓
- `1002` 是 `10020101` 的祖父级（非直接父级）
- `1002` 不是 `100301` 的父级 ✗（前缀不匹配）

### 按名称路径判定

较短路径是较长路径的前缀 → 上下级关系。

示例：
- `1002\存放中央银行款项` 是 `100201\存放中央银行款项\存放中央银行准备金存款` 的父级
  - 名称路径前缀匹配：`存放中央银行款项` = `存放中央银行款项`
- `存放同业款项` 是 `存放同业款项\存放银行业同业活期` 的父级

## 快速理解示例

给定以下数据行，如何理解：

```
编码         名称
1002         1002\存放中央银行款项
100201       100201\存放中央银行款项\存放中央银行准备金存款
100202       100202\存放中央银行款项\存放中央银行财政性存款
1011         1011\存放同业款项
101101       101101\存放同业款项\存放银行业同业活期
10110103     10110103\存放同业款项\存放银行业同业活期\存放政策性银行同业活期款项
```

解读：
1. `1002`(L1) 是资产类一级科目"存放中央银行款项"
2. `100201`(L2) 和 `100202`(L2) 都是 `1002` 的二级子科目
3. `1011`(L1) 是另一个资产类一级科目"存放同业款项"
4. `101101`(L2) 是 `1011` 下的二级科目
5. `10110103`(L3) 是 `101101` 下的三级科目，最终名称为"存放政策性银行同业活期款项"
