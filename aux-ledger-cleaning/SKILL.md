# 会计核算平台辅助明细账数据清洗

## 触发条件

当处理**会计核算平台**导出的**辅助明细账** xlsx 文件时使用，包括但不限于：
- 解析辅助明细账 Excel 文件的表头结构
- 识别并过滤汇总行（期初/本月合计/本年累计/总计）
- 合并日期列（月+日→完整日期）
- 提取机构信息
- 解析科目编码

适用于任意后续存储格式（Excel / CSV / SQLite 等）。

## 模块 1：表头识别

### 结构

```
行1: 全部为 None（空行，跳过）
行2: A2:B2 合并 = 年份（如 "2025"）
      C2~M2 = 列标题文本（各自与行3跨行合并）
行3: A3 = "月", B3 = "日"
      C3~M3 = None（合并单元格的下半部分）
```

### 合并单元格清单

| 合并范围 | 内容 | 含义 |
|---------|------|------|
| A2:B2 | `2025` | 数据年份 |
| C2:C3 | `核算账簿名称` | 列标题跨行 |
| D2:D3 | `科目名称` | 列标题跨行 |
| E2:E3 | `部门名称` | 列标题跨行 |
| F2:F3 | `来源系统名称` | 列标题跨行 |
| G2:G3 | `凭证号` | 列标题跨行 |
| H2:H3 | `摘要` | 列标题跨行 |
| I2:I3 | `对方科目` | 列标题跨行 |
| J2:J3 | `借方` | 列标题跨行 |
| K2:K3 | `贷方` | 列标题跨行 |
| L2:L3 | `方向` | 列标题跨行 |
| M2:M3 | `余额` | 列标题跨行 |

### 解析算法

```python
def parse_header(ws):
    year = str(ws.cell(row=2, column=1).value)  # A2 = 年份
    data_start_row = 4
    return year, data_start_row
```

## 模块 2：日期合并

### 规则

- **年份**：从 A2 单元格读取（值为字符串或数字，如 `2025` 或 `"2025"`）
- **月**：A 列（第 4 行起），可能为字符串 `"06"` 或数字 `6`
- **日**：B 列（第 4 行起），可能为字符串 `"30"` 或数字 `30`
- **合并结果**：`yyyy-MM-dd` 格式，如 `2025-06-30`

### 特殊情况

| A 列 | B 列 | 结果 | 说明 |
|------|------|------|------|
| `06` | `30` | `2025-06-30` | 正常明细行 |
| `6` | `3` | `2025-06-03` | 数字格式，补零 |
| 空/`""` | 空/`""` | `None` | 汇总行，不生成日期 |

### 解析算法

```python
def merge_date(year: str, month_val, day_val) -> str | None:
    if not month_val or not day_val:
        return None
    month = str(int(month_val)).zfill(2)
    day = str(int(day_val)).zfill(2)
    return f"{year}-{month}-{day}"
```

## 模块 3：行类型识别

### 判断依据

通过 **H 列（摘要）** 的值进行分类：

| H 列精确值 | 行类型 | A/B 列状态 | C 列状态 | 处理 |
|-----------|--------|-----------|---------|------|
| `期初` | 期初余额行 | 空 | 有值（账簿名） | 丢弃 |
| `本月合计` | 月度合计行 | 空 | 有值（账簿名） | 丢弃 |
| `本年累计` | 年度累计行 | 空 | 有值（账簿名） | 丢弃 |
| `总计` | 文件总计行 | 空 | 空 | 丢弃 |
| 其他任意值 | **明细数据行** | 有值（月/日） | 有值（账簿名） | **保留** |

### 文件内部分组结构

每个文件内按「核算账簿」交替分段：

```
[期初]      → 账簿 A 的期初余额
[明细...]    → 账簿 A 的交易明细（多行）
[本月合计]   → 账簿 A 本月合计
[本年累计]   → 账簿 A 本年累计
[期初]      → 账簿 B 的期初余额
[明细...]    → 账簿 B 的交易明细
[本月合计]   → 账簿 B 本月合计
[本年累计]   → 账簿 B 本年累计
...（重复）
[总计]      → 全文件总计（最后一行）
```

同一账簿可能出现多次分段（按月重复出现 本月合计 + 本年累计）。

### 解析算法

```python
SUMMARY_KEYWORDS = {'期初', '本月合计', '本年累计', '总计'}

def classify_row(h_value) -> str:
    """返回 'detail' 或具体汇总类型"""
    if h_value in SUMMARY_KEYWORDS:
        return h_value
    return 'detail'

def is_detail_row(h_value) -> bool:
    return h_value not in SUMMARY_KEYWORDS
```

## 模块 4：机构信息提取

### 目录命名规则

标准格式：`{7位机构代码}-{4位序号}{机构中文名}{后缀}`

示例：
```
0200000-0001内蒙古农商银行呼和浩特中心支行汇总基准账簿
0300000-0001内蒙古农商银行包头中心支行汇总基
1000000-0001内蒙古农商银行鄂尔多斯中支汇
```

### 提取规则

```python
import re

def extract_org_info(dir_name: str) -> tuple[str, str]:
    """从目录名提取 (机构代码, 机构名称)"""
    m = re.match(r'^(\d{7})-\d{4}(.+?)(?:汇总.*)?$', dir_name)
    if m:
        return m.group(1), m.group(2)
    return None, dir_name
```

### 特殊目录映射

| 目录名 | 机构代码 | 机构名称 |
|--------|---------|---------|
| `内蒙古农商银行赤峰中心支行汇总` | `0700000` | `内蒙古农商银行赤峰中心支行` |
| `辅助明细2/0700000-...` | 从子目录提取 | 从子目录提取 |
| `辅助明细2_2/1500000-...` | 从子目录提取 | 从子目录提取 |

### 路径解析逻辑

```python
def resolve_org_from_path(file_path: str, base_dir: str) -> tuple[str, str]:
    """根据文件相对路径解析机构信息"""
    rel = os.path.relpath(file_path, base_dir)
    parts = rel.split(os.sep)

    if parts[0] in ('辅助明细2', '辅助明细2_2'):
        org_dir = parts[1]  # 子目录才是机构目录
    elif parts[0] == '内蒙古农商银行赤峰中心支行汇总':
        return '0700000', '内蒙古农商银行赤峰中心支行'
    else:
        org_dir = parts[0]

    return extract_org_info(org_dir)
```

## 模块 5：科目编码解析

### 前置依赖

集成 `account-code-understanding` Skill 中的编码规则。

### D 列格式

```
{完整编码}\{一级科目名}\{二级科目名}\...\{当前科目名}
```

示例：
```
60110106\利息收入\农户贷款利息收入\10万元以下逾期农户贷款利息收入
642102\手续费及佣金支出\结算业务手续费支出
670201\信用减值损失\存放同业减值损失
```

### 解析规则

```python
def parse_account(subject_name: str) -> tuple[str, str, str, str]:
    """
    解析科目名称列，返回 (完整编码, 一级科目编码, 一级科目名称, 科目全称)

    示例:
      输入: "60110106\\利息收入\\农户贷款利息收入\\10万元以下逾期农户贷款利息收入"
      输出: ("60110106", "6011", "利息收入", 原值)
    """
    if not subject_name:
        return ('', '', '', '')

    parts = subject_name.split('\\')
    full_code = parts[0]
    level1_code = full_code[:4] if len(full_code) >= 4 else full_code
    level1_name = parts[1] if len(parts) > 1 else ''
    return (full_code, level1_code, level1_name, subject_name)
```

## 数据质量注意事项

1. **借方/贷方空值**：空字符串 `""` 或 `None`，非零值，写入时应保持为空
2. **负数金额**：贷方列可能出现负数（如 `-53.06`），属于红字冲销，应如实保留
3. **月份列可能包含前导零**：`"06"` 与 `6` 应视为相同
4. **科目编码长度不固定**：4-10 位均可能出现，但一级科目固定取前 4 位
5. **同一文件可能包含多个核算账簿**：数据行的 C 列值会在账簿切换时变化
